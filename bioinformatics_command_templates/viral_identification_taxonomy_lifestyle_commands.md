# Viral sequence identification, clustering, taxonomic assignment and lifestyle prediction

This file documents representative command structures for viral sequence identification, CheckV-based curation, vOTU clustering, taxonomic assignment and lifestyle prediction.

The full LMVD construction was performed on 4,017 livestock manure metagenomes using large-scale HPC workflows. The commands below record the key tools, parameters and filtering logic used in the analysis. They are not intended to constitute an end-to-end executable workflow for all 4,017 samples, and file names and paths should be specified according to the local directory structure.

## 1. Input structure

Each metagenomic sample was assembled separately. Viral sequence identification was performed on the assembled contigs from each sample.

Representative input variables for one sample:

```bash
SAMPLE="Sample_001"
ASSEMBLY="assemblies/${SAMPLE}.contigs.fa"
THREADS=28
```

Representative input:

```text
assemblies/
└── Sample_001.contigs.fa
```

Representative output directory for one sample:

```text
viral_prediction/
└── Sample_001/
    ├── vs2-pass1/
    ├── checkv/
    └── curated/
```

## 2. Viral sequence identification using VirSorter2

Putative viral sequences longer than 5 kb were identified using VirSorter2. The `--keep-original-seq` option was used to retain circular and near-complete viral contigs. Only dsDNA phage and ssDNA groups were included.

```bash
mkdir -p viral_prediction/${SAMPLE}

virsorter run \
  --keep-original-seq \
  -i ${ASSEMBLY} \
  -w viral_prediction/${SAMPLE}/vs2-pass1 \
  --include-groups dsDNAphage,ssDNA \
  --min-length 5000 \
  --min-score 0.5 \
  -j ${THREADS} \
  all
```

Main VirSorter2 output:

```text
viral_prediction/${SAMPLE}/vs2-pass1/final-viral-combined.fa
viral_prediction/${SAMPLE}/vs2-pass1/final-viral-score.tsv
```

## 3. CheckV quality control and prophage boundary trimming

VirSorter2 output was processed using CheckV to assess viral sequence quality and to trim potential host-derived regions flanking prophages.

```bash
checkv end_to_end \
  viral_prediction/${SAMPLE}/vs2-pass1/final-viral-combined.fa \
  viral_prediction/${SAMPLE}/checkv \
  -t ${THREADS} \
  -d /path/to/checkv_database
```

The retained CheckV viral and proviral sequences were combined:

```bash
cat \
  viral_prediction/${SAMPLE}/checkv/proviruses.fna \
  viral_prediction/${SAMPLE}/checkv/viruses.fna \
  > viral_prediction/${SAMPLE}/checkv/combined.fna
```

## 4. SOP-based curation of putative viral contigs

Contigs were filtered according to the Viral Sequence Identification SOP. The following empirical categories were used:

```text
Keep1: viral_gene > 0
Keep2: viral_gene = 0 AND (host_gene = 0 OR score >= 0.95 OR hallmark_gene > 2)
Manual-check category: viral_gene = 0 AND host_gene = 1 AND length >= 10 kb, if not already in Keep1 or Keep2
Discard: remaining sequences
```

In this study, only contigs classified as Keep1 or Keep2 were retained as putative metagenomic viral contigs for downstream analyses.

The filtering table was generated by merging the VirSorter2 score table and the CheckV contamination table:

```bash
python scripts/merge_vs2_checkv_tables.py \
  --vs2 viral_prediction/${SAMPLE}/vs2-pass1/final-viral-score.tsv \
  --checkv viral_prediction/${SAMPLE}/checkv/contamination.tsv \
  --out viral_prediction/${SAMPLE}/curated/${SAMPLE}.vs2_checkv_merged.tsv
```

Representative filtering command:

```bash
python scripts/filter_keep1_keep2.py \
  --metadata viral_prediction/${SAMPLE}/curated/${SAMPLE}.vs2_checkv_merged.tsv \
  --fasta viral_prediction/${SAMPLE}/checkv/combined.fna \
  --out-fasta viral_prediction/${SAMPLE}/curated/${SAMPLE}.VC.keep1_keep2.fa \
  --out-table viral_prediction/${SAMPLE}/curated/${SAMPLE}.VC.keep1_keep2.tsv
```

A minimum length threshold of 5 kb was enforced after curation:

```bash
seqkit seq \
  -m 5000 \
  viral_prediction/${SAMPLE}/curated/${SAMPLE}.VC.keep1_keep2.fa \
  > viral_prediction/${SAMPLE}/curated/${SAMPLE}.VC.keep1_keep2.min5kb.fa
```

Curated viral contigs from all samples were then combined:

```bash
mkdir -p LMVD_vOTU_clustering

cat viral_prediction/*/curated/*.VC.keep1_keep2.min5kb.fa \
  > LMVD_vOTU_clustering/all_curated_VCs.min5kb.fa
```

## 5. vOTU clustering

Curated viral contigs were clustered into vOTUs at the species level using 95% average nucleotide identity over 85% of the shortest contig.

The clustering workflow used `anicalc.py` and `aniclust.py` from the CheckV repository.

```bash
VC_FASTA="LMVD_vOTU_clustering/all_curated_VCs.min5kb.fa"

makeblastdb \
  -in ${VC_FASTA} \
  -dbtype nucl \
  -out LMVD_vOTU_clustering/all_curated_VCs.min5kb.db

blastn \
  -query ${VC_FASTA} \
  -db LMVD_vOTU_clustering/all_curated_VCs.min5kb.db \
  -outfmt '6 std qlen slen' \
  -max_target_seqs 10000 \
  -out LMVD_vOTU_clustering/all_curated_VCs.min5kb.blast.tsv \
  -num_threads 32

python /path/to/checkv/scripts/anicalc.py \
  -i LMVD_vOTU_clustering/all_curated_VCs.min5kb.blast.tsv \
  -o LMVD_vOTU_clustering/all_curated_VCs.min5kb.ani.tsv

python /path/to/checkv/scripts/aniclust.py \
  --fna ${VC_FASTA} \
  --ani LMVD_vOTU_clustering/all_curated_VCs.min5kb.ani.tsv \
  --out LMVD_vOTU_clustering/all_curated_VCs.min5kb.vOTU_clusters.tsv \
  --min_ani 95 \
  --min_tcov 85 \
  --min_qcov 0
```

Main outputs:

```text
LMVD_vOTU_clustering/all_curated_VCs.min5kb.fa
LMVD_vOTU_clustering/all_curated_VCs.min5kb.vOTU_clusters.tsv
```

## 6. Taxonomic assignment of viral genomes

Taxonomic assignment was performed using geNomad and PhaGCN2. geNomad was run in `end-to-end` mode. PhaGCN2 was run using the official command-line interface, which takes a contig FASTA file and a minimum contig-length parameter. The PhaGCN2 output is written as `final_prediction.csv` in the running directory.

### 6.1 geNomad

Representative command structure for one sample or one FASTA file:

```bash
INPUT_FASTA="LMVD_vOTU_clustering/all_curated_VCs.min5kb.fa"
GENOMAD_OUT="taxonomy/geNomad/all_curated_VCs"
GENOMAD_DB="/path/to/genomad_db"

genomad end-to-end \
  --cleanup \
  -t 32 \
  ${INPUT_FASTA} \
  ${GENOMAD_OUT} \
  ${GENOMAD_DB}
```

### 6.2 PhaGCN2

The PhaGCN2 command-line interface takes a contig FASTA file and a minimum contig-length parameter:

```bash
python run_Speed_up.py --contigs contigs.fa --len 8000
```

Here, `--contigs` is the input contig FASTA file and `--len` is the minimum contig length for prediction. The default minimum length is 8000 bp. Because the retained viral contigs were longer than 5 kb, `--len` should be set according to the cutoff used for PhaGCN2 classification. If all retained viral contigs longer than 5 kb were classified, use `--len 5000`; if the default threshold was retained, use `--len 8000` or omit the parameter.

Representative command structure for classifying curated viral contigs longer than 5 kb:

```bash
INPUT_FASTA="LMVD_vOTU_clustering/all_curated_VCs.min5kb.fa"
PHAGCN_DIR="taxonomy/PhaGCN2"

mkdir -p ${PHAGCN_DIR}
cd ${PHAGCN_DIR}

conda activate phagcn2
export MKL_SERVICE_FORCE_INTEL=1

python /path/to/PhaGCN2.0/run_Speed_up.py \
  --contigs ../../${INPUT_FASTA} \
  --len 5000

mv final_prediction.csv LMVD_PhaGCN2_prediction.csv
cd -
```

Main PhaGCN2 output:

```text
taxonomy/PhaGCN2/LMVD_PhaGCN2_prediction.csv
```

If the default minimum length was used instead, replace `--len 5000` with `--len 8000`.

### 6.3 Merging taxonomic assignments

Taxonomic assignments from geNomad and PhaGCN2 were merged according to the taxonomic-priority rule described in the manuscript.

Representative command structure:

```bash
python scripts/merge_viral_taxonomy.py \
  --genomad taxonomy/geNomad/all_curated_VCs \
  --phagcn taxonomy/PhaGCN2/LMVD_PhaGCN2_prediction.csv \
  --clusters LMVD_vOTU_clustering/all_curated_VCs.min5kb.vOTU_clusters.tsv \
  --out taxonomy/LMVD_vOTU_taxonomy.tsv
```

Main output:

```text
taxonomy/LMVD_vOTU_taxonomy.tsv
```

## 7. Lifestyle prediction using PhaTYP

Lifestyle prediction was performed using PhaTYP. The official PhaTYP workflow uses two steps: preprocessing the input contig FASTA file and then running `PhaTYP.py` on the intermediate folder. The output CSV file contains contig names, lifestyle prediction and prediction score.

```bash
INPUT_FASTA="LMVD_vOTU_clustering/all_curated_VCs.min5kb.fa"
PHATYP_DIR="lifestyle/PhaTYP"
MID_DIR="${PHATYP_DIR}/midfolder"
OUT_CSV="${PHATYP_DIR}/LMVD_PhaTYP_lifestyle_prediction.csv"

mkdir -p ${PHATYP_DIR}

conda activate phatyp

python /path/to/PhaTYP/preprocessing.py \
  --contigs ${INPUT_FASTA} \
  --len 5000 \
  --midfolder ${MID_DIR}

python /path/to/PhaTYP/PhaTYP.py \
  --out ${OUT_CSV} \
  --midfolder ${MID_DIR}
```

The PhaTYP default minimum length is 3000 bp. Because the curated viral contigs used here were longer than 5 kb, `--len 5000` is shown to match the retained viral-contig cutoff. If the default threshold was used, omit `--len 5000`.

Main output:

```text
lifestyle/PhaTYP/LMVD_PhaTYP_lifestyle_prediction.csv
```

## 8. Scope notes

1. The commands above document representative tool calls and filtering logic rather than a complete executable reconstruction of the 4,017-sample HPC workflow.
2. Local paths, sample names and wrapper scripts should be specified according to the computing environment.
3. The key viral sequence curation decisions were: VirSorter2 identification, CheckV quality control and prophage trimming, retention of Keep1 and Keep2 contigs, and final filtering to sequences longer than 5 kb.
4. vOTU clustering was performed at 95% ANI over 85% of the shortest contig.
5. geNomad and PhaGCN2 were both used for taxonomic assignment, and PhaTYP was used for lifestyle prediction.
