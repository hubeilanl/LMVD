# MAG reconstruction, mOTU profiling, taxonomy and growth-rate estimation

This file documents representative command structures for metagenome-assembled genome (MAG) reconstruction, MAG quality assessment, mOTU dereplication, taxonomic classification, abundance profiling and growth-rate estimation.

The full analysis was performed on large-scale metagenomic datasets using HPC workflows. The commands below describe the key tools, parameters and input/output logic at the level of representative samples or analysis stages. They are not intended to constitute an end-to-end executable workflow for all 4,017 samples.

## 1. Per-sample inputs and directory structure

Representative variables for one metagenomic sample:

```bash
SAMPLE="Sample_001"
THREADS=8
MEM_GB=72

ASSEMBLY="all_contigs/${SAMPLE}.fa"
READ1="1-Trim/PE/${SAMPLE}_1.fastq.gz"
READ2="1-Trim/PE/${SAMPLE}_2.fastq.gz"
```

Representative working directories:

```text
all_contigs/                 # assembled contigs
1-Trim/PE/                   # quality-filtered paired-end reads
MAG_reconstruction/          # metaWRAP binning and refinement output
MAGs/                        # refined MAGs
MAG_quality/                 # CheckM2 output
mOTU_dereplication/          # dRep output
MAG_taxonomy/                # GTDB-Tk output
MAG_abundance/               # CoverM output
MAG_growth/                  # GRiD output
```

## 2. MAG reconstruction using metaWRAP

MAGs were reconstructed for each metagenome using metaWRAP with MetaBAT2 and MaxBin2.

```bash
mkdir -p MAG_reconstruction

metawrap binning \
  -o MAG_reconstruction/${SAMPLE}_Bin \
  -t ${THREADS} \
  -m ${MEM_GB} \
  -a ${ASSEMBLY} \
  --metabat2 \
  --maxbin2 \
  ${READ1} \
  ${READ2}
```

Main outputs:

```text
MAG_reconstruction/${SAMPLE}_Bin/metabat2_bins/
MAG_reconstruction/${SAMPLE}_Bin/maxbin2_bins/
```

## 3. Bin refinement

Bins generated by MetaBAT2 and MaxBin2 were refined using metaWRAP. In this workflow, bins with completeness ≥70% and contamination ≤10% were retained after refinement.

```bash
metawrap bin_refinement \
  -o MAG_reconstruction/${SAMPLE}_Bin \
  -t ${THREADS} \
  -m ${MEM_GB} \
  -A MAG_reconstruction/${SAMPLE}_Bin/metabat2_bins/ \
  -B MAG_reconstruction/${SAMPLE}_Bin/maxbin2_bins/ \
  -c 70 \
  -x 10
```

Main output:

```text
MAG_reconstruction/${SAMPLE}_Bin/metawrap_70_10_bins/
```

Copy refined MAGs into a standardized sample-specific directory:

```bash
mkdir -p MAGs/${SAMPLE}

cp MAG_reconstruction/${SAMPLE}_Bin/metawrap_70_10_bins/*.fa \
  MAGs/${SAMPLE}/
```

Intermediate metaWRAP files can be removed after the refined bins have been verified:

```bash
rm -rf MAG_reconstruction/${SAMPLE}_Bin/work_files
rm -rf MAG_reconstruction/${SAMPLE}_Bin/figures
rm -rf MAG_reconstruction/${SAMPLE}_Bin/maxbin2_*
rm -rf MAG_reconstruction/${SAMPLE}_Bin/metabat2_*
rm -rf MAG_reconstruction/${SAMPLE}_Bin/metawrap_70_10_bins.contigs
```

## 4. MAG quality assessment using CheckM2

CheckM2 was used to estimate MAG completeness and contamination.

```bash
mkdir -p MAG_quality/checkm2/${SAMPLE}

checkm2 predict \
  --threads 32 \
  -x fa \
  --input MAGs/${SAMPLE} \
  --output-directory MAG_quality/checkm2/${SAMPLE}
```

Main output:

```text
MAG_quality/checkm2/${SAMPLE}/quality_report.tsv
```

## 5. Preparing input files for dRep

For dRep dereplication, refined MAGs from all samples were combined into one directory.

```bash
mkdir -p mOTU_dereplication/all_MAGs

find MAGs/ -name "*.fa" -exec cp {} mOTU_dereplication/all_MAGs/ \;
```

dRep also requires a genome information file containing genome name, completeness and contamination.

Required format:

```text
genome,completeness,contamination
Sample_001_bin.1.fa,92.35,1.28
Sample_001_bin.2.fa,74.83,0.36
Sample_002_bin.4.fa,90.00,3.93
```

Representative command structure for generating this table from CheckM2 outputs:

```bash
mkdir -p mOTU_dereplication/tables

echo "genome,completeness,contamination" \
  > mOTU_dereplication/tables/all_MAGs.checkm2.genomeInfo.csv

for REPORT in MAG_quality/checkm2/*/quality_report.tsv
do
  awk 'BEGIN{FS=OFS="\t"} NR==1{next} {print $1","$2","$3}' ${REPORT} \
    >> mOTU_dereplication/tables/all_MAGs.checkm2.genomeInfo.csv
done
```

If CheckM2 output columns differ among versions, the parsing command should be revised so that the final CSV contains the three required columns: `genome`, `completeness` and `contamination`.

## 6. mOTU dereplication using dRep

dRep was used to dereplicate MAGs into mOTUs. The dereplicated mOTU set was used to reduce redundancy for downstream analyses such as virus–host prediction. ARG annotation and AR-MAG identification were performed on the original, non-dereplicated MAG set.

```bash
mkdir -p mOTU_dereplication/dRep

dRep dereplicate \
  mOTU_dereplication/dRep/all_MAGs_dRep \
  --genomeInfo mOTU_dereplication/tables/all_MAGs.checkm2.genomeInfo.csv \
  -g mOTU_dereplication/all_MAGs/*.fa \
  -comp 50 \
  -con 10 \
  -p 32 \
  --S_ani 0.95 \
  --S_algorithm fastANI \
  -nc 0.3
```

Key outputs:

```text
mOTU_dereplication/dRep/all_MAGs_dRep/dereplicated_genomes/
mOTU_dereplication/dRep/all_MAGs_dRep/data_tables/
```

The dRep data tables were used to assign each MAG to a secondary cluster and to identify the representative genome for each mOTU.

## 7. MAG abundance profiling using CoverM

Reads from each sample were mapped back to the MAGs recovered from the corresponding sample. The following read-alignment filters were used:

```text
--min-read-aligned-percent 75
--min-read-aligned-length 75
--min-read-percent-identity 80
```

The `trimmed_mean` metric was used for coverage estimation.

```bash
mkdir -p MAG_abundance/coverm

coverm genome \
  --coupled ${READ1} ${READ2} \
  --genome-fasta-directory MAGs/${SAMPLE} \
  -o MAG_abundance/coverm/${SAMPLE}.MAG_coverage.tsv \
  -x fa \
  -m trimmed_mean \
  --min-read-aligned-percent 75 \
  --min-read-aligned-length 75 \
  --min-read-percent-identity 80
```

Command structure for calculating both arithmetic mean and trimmed mean coverage:

```bash
coverm genome \
  --coupled ${READ1} ${READ2} \
  --genome-fasta-directory MAGs/${SAMPLE} \
  -o MAG_abundance/coverm/${SAMPLE}.MAG_coverage.mean_and_trimmed_mean.tsv \
  -x fa \
  -m mean \
  -m trimmed_mean \
  --min-read-aligned-percent 75 \
  --min-read-aligned-length 75 \
  --min-read-percent-identity 80
```

MAG coverage values were converted to cell-normalized abundance using the estimated microbial cell number for each sample. mOTU abundance was calculated as the mean cell-normalized abundance of MAGs assigned to the same mOTU cluster.

## 8. MAG taxonomic classification using GTDB-Tk

GTDB-Tk was used to classify MAGs taxonomically.

```bash
mkdir -p MAG_taxonomy/gtdbtk/${SAMPLE}

gtdbtk classify_wf \
  --genome_dir MAGs/${SAMPLE} \
  --out_dir MAG_taxonomy/gtdbtk/${SAMPLE} \
  --mash_db /path/to/mash_dir \
  --cpus 32 \
  -x fa \
  --prefix ${SAMPLE}
```

Main outputs:

```text
MAG_taxonomy/gtdbtk/${SAMPLE}/${SAMPLE}.bac120.summary.tsv
MAG_taxonomy/gtdbtk/${SAMPLE}/${SAMPLE}.ar53.summary.tsv
```

## 9. Optional GTDB-Tk marker-gene tree construction

The following commands document the marker-gene tree construction structure.

### 9.1 Identify marker genes

```bash
gtdbtk identify \
  --genome_dir MAGs/${SAMPLE} \
  --out_dir MAG_taxonomy/gtdbtk_tree/${SAMPLE}_identify \
  --cpus 32 \
  -x fa \
  --prefix ${SAMPLE}
```

### 9.2 Align marker genes

```bash
gtdbtk align \
  --identify_dir MAG_taxonomy/gtdbtk_tree/${SAMPLE}_identify \
  --out_dir MAG_taxonomy/gtdbtk_tree/${SAMPLE}_align \
  --skip_gtdb_refs \
  --cpus 32 \
  --prefix ${SAMPLE}
```

### 9.3 Infer a phylogenetic tree

```bash
gtdbtk infer \
  --msa_file MAG_taxonomy/gtdbtk_tree/${SAMPLE}_align/align/${SAMPLE}.bac120.user_msa.fasta.gz \
  --out_dir MAG_taxonomy/gtdbtk_tree/${SAMPLE}_infer \
  --prot_model LG \
  --gamma \
  --prefix ${SAMPLE} \
  --cpus 32
```

### 9.4 Convert the tree for iTOL visualization

```bash
gtdbtk convert_to_itol \
  --input MAG_taxonomy/gtdbtk_tree/${SAMPLE}_infer/${SAMPLE}.bac120.tree \
  --output MAG_taxonomy/gtdbtk_tree/${SAMPLE}_infer/${SAMPLE}.itol.tree
```

## 10. MAG growth-rate estimation using GRiD

GRiD was used to estimate MAG-level growth rates. Because the metagenomic reads were paired-end reads, reads were first mapped to each MAG using Bowtie2. The resulting SAM files were then used as input for `grid single`.

```bash
mkdir -p MAG_growth/GRiD/${SAMPLE}/sam
mkdir -p MAG_growth/GRiD/${SAMPLE}/grid
mkdir -p MAG_growth/GRiD/${SAMPLE}/summary

for MAG in MAGs/${SAMPLE}/*.fa
do
  MAG_FILE=$(basename ${MAG})
  MAG_ID=${MAG_FILE%.*}

  SAM_DIR="MAG_growth/GRiD/${SAMPLE}/sam/${MAG_ID}"
  GRID_DIR="MAG_growth/GRiD/${SAMPLE}/grid/${MAG_ID}"

  mkdir -p ${SAM_DIR}
  mkdir -p ${GRID_DIR}

  bowtie2-build \
    --threads ${THREADS} \
    ${MAG} \
    ${SAM_DIR}/${MAG_ID}

  bowtie2 \
    -p ${THREADS} \
    -x ${SAM_DIR}/${MAG_ID} \
    -1 ${READ1} \
    -2 ${READ2} \
    -S ${SAM_DIR}/${MAG_ID}.sam \
    --no-unal \
    --quiet

  grid single \
    -r ${SAM_DIR} \
    -e sam \
    -g ${MAG} \
    -o ${GRID_DIR} \
    -n ${THREADS}

  rm -rf ${SAM_DIR}
done
```

Summarize per-MAG GRiD outputs for one sample:

```bash
find MAG_growth/GRiD/${SAMPLE}/grid -name "*.GRiD.txt" | sort \
  > MAG_growth/GRiD/${SAMPLE}/summary/${SAMPLE}.grid_files.txt

awk 'FNR==1 && NR==1 {print; next} FNR>1 {print}' \
  $(cat MAG_growth/GRiD/${SAMPLE}/summary/${SAMPLE}.grid_files.txt) \
  > MAG_growth/GRiD/${SAMPLE}/summary/${SAMPLE}.GRiD.summary.tsv

rm -f MAG_growth/GRiD/${SAMPLE}/summary/${SAMPLE}.grid_files.txt
```

If the input reads are in FASTA format rather than FASTQ format, add `-f` to the `bowtie2` command. For standard FASTQ or FASTQ.GZ input, `-f` should not be used.

mOTU-level growth rates were calculated as the mean GRiD value of MAGs assigned to the same mOTU cluster.

```text

MAG_quality/checkm2/                       # MAG completeness and contamination

MAG_taxonomy/gtdbtk/                       # GTDB-Tk taxonomic assignments

```

## 12. Scope notes

1. The commands above document the main workflow components and parameters used for MAG-related analyses.
2. metaWRAP binning used MetaBAT2 and MaxBin2.
3. Bin refinement retained bins with completeness ≥70% and contamination ≤10%.
4. CheckM2 was used for final MAG quality assessment.
5. dRep dereplication used `-comp 50`, `-con 10`, `--S_ani 0.95`, `--S_algorithm fastANI` and `-nc 0.3`.
6. dRep was used to generate non-redundant mOTUs, whereas AR-MAG identification was performed on the original, non-dereplicated MAG set.
7. CoverM abundance estimation used `trimmed_mean` and the read-alignment filters listed above.
8. GRiD was used for MAG-level growth-rate estimation, and mOTU-level growth rates were summarized from MAG-level GRiD values.
